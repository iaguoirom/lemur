**目的**: 未知の環境（湯気、モーションブラー、重なり）で崩壊した座標を修復し、精度を向上させる。

## 抽出アルゴリズム：何をもって間違いとするか

`extract_outlier_frames` 関数でAIがエラーを抽出する際、以下の3つの論理的基準（アルゴリズム）を選択できる。

| **アルゴリズム**      | **判別基準（ロジック）**                                                               | **有効なケース**           |
| --------------- | ---------------------------------------------------------------------------- | -------------------- |
| **jump**        | 前後のフレーム間のユークリッド距離 $d = \sqrt{(x_2-x_1)^2 + (y_2-y_1)^2}$ が閾値（epsilon）を超えた場合。 | 点が背景にワープする現象。        |
| **likelihood**  | 尤度が設定値（p-bound）を下回った場合。                                                      | 点が点滅したり、消失したりする現象。   |
| **uncertainty** | 予測の分散が大きいフレームを抽出する。                                                          | 複数匹の個体が重なり、判定が曖昧な場合。 |

## 3. 手動修正

行うべき作業

 **座標の修正**: 
 - AIが背景のノイズで勘違いして打った点を、正しい位置へ戻す。

 **見えない部分の処理**:
- 木の陰などに隠れて見えない部位の点は、無理に打たずに削除するか、そのまま放置する。

## 4. 学習の再統合（Merge and Re-train）

修正が終わった後の処理↓

$$\text{既存の学習データ} + \text{今回修正した20枚} = \text{新・学習用データセット}$$

1. **`merge_datasets`**: 修正した座標を、プロジェクト全体の `.h5` データベースへ統合する。

2. **`train_network`**: 更新されたデータセットを用いて、ネットワークを再学習させる。

- **ポイント**: 0から学習し直すのではなく、既存の重みを引き継いで「差分」だけを学習させることで、短時間で精度を向上させる。


## 5. 評価指標

修正後のモデルを評価する際、以下の2点を確認する。

- **RMSE（二乗平均平方根誤差）の低下**: 人間が打った正解と、AIの予測の距離が縮まっているか。
- **軌跡（Trajectory）**: 速度グラフを表示した際、不自然な線が消え、滑らかな曲線を描いているか。
