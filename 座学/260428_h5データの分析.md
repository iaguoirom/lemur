Jerkグラフより、サルの鼻があり得ない振動をしていることになる。
このような状態になるとき、次の手法を導入する。

サビツキーゴーレイ法（SG法）
奇数の長さの部分区間に定数関数を当てはめ、中央の値を平滑化された値として採用することでノイズを低減させる手法である。

```
import numpy as np
import pandas as pd
from scipy.signal import savgol_filter
import matplotlib.pyplot as plt

# --- 1. パラメータ設定（ここだけ調整する） ---
TARGET_PART = 'nose'  # 解析したい部位
THRESHOLD = 0.9       # 尤度の閾値
W_LEN = 51            # スムージングの窓幅（奇数。ガタつくなら 71, 91 と増やす。奇数にする）
P_ORD = 3             # 近似する次数の高さ

# --- 2. データの「掃除」と「正規化」 ---
df_work = df.copy()
if df_work.columns.nlevels > 2:
    df_work.columns = df_work.columns.droplevel(0) # Scorer削除

# 尤度の低い値をNaNに変え、線形補間し、冒頭の欠損をbfillで埋める
low_conf = df_work[TARGET_PART]['likelihood'] < THRESHOLD
df_work.loc[low_conf, (TARGET_PART, ['x', 'y'])] = np.nan
df_clean = df_work.interpolate().bfill()

# --- 3. データの「研磨（SG法）」 ---
# 座標データそのものを滑らかにする
x_s = savgol_filter(df_clean[TARGET_PART]['x'], W_LEN, P_ORD)
y_s = savgol_filter(df_clean[TARGET_PART]['y'], W_LEN, P_ORD)

# --- 4. 物理量の算出（NumPyの勾配関数 gradient を使用） ---
# 速度 (Velocity) の大きさ
vx = np.gradient(x_s)
vy = np.gradient(y_s)
v_s = np.sqrt(vx**2 + vy**2)

# 加速度 (Acceleration) と 躍度 (Jerk)
# diff() よりも端点の処理が安定する gradient を採用
a_s = np.gradient(v_s)
j_s = np.gradient(a_s)

# --- 5. 可視化（プロット） ---
fig, ax = plt.subplots(3, 1, figsize=(12, 12), sharex=True)

# 速度：掃除しただけのデータ(gray)と研磨後(blue)を比較
v_raw = np.sqrt(np.gradient(df_clean[TARGET_PART]['x'])**2 + np.gradient(df_clean[TARGET_PART]['y'])**2)
ax[0].plot(v_raw, color='gray', alpha=0.3, label='Raw (Cleaned)')
ax[0].plot(v_s, color='blue', label=f'Smoothed (W={W_LEN})')
ax[0].set_title(f'Velocity of {TARGET_PART}')
ax[0].set_ylabel('Pixels/frame')
ax[0].legend()

# 加速度
ax[1].plot(a_s, color='green')
ax[1].set_title('Acceleration')
ax[1].set_ylabel('$\Delta$ Velocity')

# 躍度
ax[2].plot(j_s, color='red')
ax[2].set_title('Jerk (Smoothness Index)')
ax[2].set_ylabel('$\Delta$ Acceleration')

plt.tight_layout()
plt.show()
```
